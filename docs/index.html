Engineered for Extreme Efficiency

Ditch slow shell copies. Replicate files directly at storage limits using custom Rust primitives.

📁

NTFS/ReFS Block Cloning

Utilizes Windows DeviceIoControl. Transfers on the same drive copy instantly (0ms) at metadata-only speed with zero byte overhead.

🏎️

Dynamic Rate Limiting

A centralized dynamic token-bucket rate limiter throttles bandwidth collectively across all parallel copy jobs in real time.

🛡️

On-the-Fly Verification

Computes Blake3, XXHash3, MD5, or SHA-256 checksums concurrently during the write loop, saving extra read cycles.

🫳

Native Drag & Drop

Drop files and folders directly onto the dashboard window to automatically queue them for copying instantly.

🔗

Explorer Context Menu

🎯

Smart Overwrite Logic

The CopyTej Pipeline

Behind the scenes of CopyTej's asynchronous double-buffered architecture.

1

File Read Thread

Tokio spawns an async task to pull chunks into a bounded double-buffer channel.

➔
2

On-the-Fly Hasher

Calculates Blake3 or XXHash checksums in memory as bytes flow through the pipe.

➔
3

File Write Thread

Writes bytes to the target volume asynchronously while preserving metadata attributes.

Getting Started

Install, build, and configure CopyTej on your desktop.

Build from Source

Ensure you have the Rust toolchain and Node.js installed.

# Clone the repository git clone https://github.com/SV-stark/CopyTej.git # Navigate and install dependencies cd CopyTej npm install # Launch dev server or compile production exe npm run tauri build

Single-Instance Named Pipe Integration

CopyTej launches a background named pipe server at \\.\pipe\CopyTej. Spawning a second instance (e.g. enqueuing via Explorer or terminal) serializes file specs and forwards them directly to the running instance's queue.

# Send files from CLI to active CopyTej queue copytej-app.exe "C:\Path\To\File.zip" -d "D:\Destination"
